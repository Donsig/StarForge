## Section 1: Planet Inline Rename

### Action layer (`src/hooks/useGameEngine.ts`)

New action: `renamePlanet(planetIndex: number, name: string): void`

Validation lives **in the action**, not the panel:
- Trim whitespace
- If empty after trim → no-op (name unchanged)
- Clamp to 30 characters max
- Mutates `state.planets[planetIndex].name`, calls `saveState`, triggers re-render

No state migration — `PlanetState.name` already exists.

### Context (`src/context/GameContext.tsx`)

Add `renamePlanet: (planetIndex: number, name: string) => void` to `GameContextType`.
Update `src/test/test-utils.tsx` and `src/panels/__tests__/AdminPanel.test.tsx` mock providers with a no-op stub.

### UI (`src/panels/OverviewPanel.tsx`)

Replace the planet name `<span>` (line 38) with an inline edit component:

- Default state: planet name text + `✏` button with `aria-label="Rename planet"`
- Click `✏` button → `<input>` with current name, `autoFocus`, max 30 chars
- **Confirm:** `Enter` key or blur → call `renamePlanet()` → exit edit mode
- **Cancel:** `Escape` key → discard draft, exit edit mode (no save)
- **Planet switch:** `useEffect` watching `activePlanetIndex` → exit edit mode, discard draft
- **Double-commit guard:** use a `committed` ref, set on first confirm, cleared on edit open — prevents Enter+blur firing twice

### IRN effect — single shared function

**Scope clarification:** IRN affects **duration only**. Building/research prerequisites remain source/active-planet checks. `sourcePlanetIndex` retains its meaning as the paying/refund planet — cancel refunds to that planet regardless of which labs contributed to timing.

Export from `src/engine/BuildQueue.ts`:
```ts
export function effectiveResearchLabLevel(state: GameState, item: QueueItem): number {
  const irnLevel = state.research.intergalacticResearchNetwork ?? 0;
  if (irnLevel === 0) {
    // unchanged behaviour — source planet lab only
    const src =
      state.planets[item.sourcePlanetIndex ?? state.activePlanetIndex] ??
      state.planets[state.activePlanetIndex] ??
      state.planets[0];
    return src?.buildings.researchLab ?? 0;
  }
  // top (irnLevel + 1) research lab levels globally
  const topN = irnLevel + 1;
  return state.planets
    .map((p) => p.buildings.researchLab)
    .sort((a, b) => b - a)
    .slice(0, topN)
    .reduce((sum, lvl) => sum + lvl, 0);
}
```

1. `BuildQueue.startResearch()` — `src/engine/BuildQueue.ts:186`
2. `BuildQueue.processTick()` next-item reschedule — `src/engine/BuildQueue.ts:276`
3. God-mode fast-complete next-item reschedule — `src/hooks/useGameEngine.ts:1027`
4. Offline catch-up reschedule — `src/engine/StateManager.ts:425`
